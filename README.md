# Architect

> Idea → frozen Blueprint → strict per-service docs tree.

Architect is a CLI that turns a human-authored product idea into a complete, executable documentation foundation. A high-intelligence model (Claude Opus by default) forges the master Blueprint; a fast-execution tier fans out per-service docs in parallel; a deterministic verifier guarantees no documentation drift.

## Why

Most AI-assisted projects suffer from documentation entropy: scattered READMEs, duplicated explanations, AI essays buried in source files, scope creep that quietly mutates the original idea. Architect treats the Blueprint as a contract — nothing exists in `docs/` unless the Blueprint mandated it, and every service gets the same eight-document spine.

The product idea (the **Spark**) is human-only. The Blueprint is forged by a high-intelligence model from the Spark. Everything downstream is execution.

## Quickstart

```bash
# Configure a provider — at least one of:
export ANTHROPIC_API_KEY=…
export OPENAI_API_KEY=…
export DEEPSEEK_API_KEY=…
export OPENROUTER_API_KEY=…       # used as fallback if a tier-mapped key is missing

# Optional: web research
export FIRECRAWL_API_KEY=…

# Sanity check
architect doctor

# End-to-end on a complete idea
architect new path/to/idea.md --out ./my-project

# End-to-end on a rough idea (interactive brainstorm)
architect new --brainstorm --out ./my-project

# Validate a tree
architect verify --out ./my-project
```

## Pipeline

| Phase | Model tier | Output |
|---|---|---|
| 0. Spark capture (optional brainstorm) | strategic | `docs/spark.md` |
| 1. Blueprint forge | strategic | `docs/blueprint.draft.md` |
| 2. QA attack review (5 perspectives) | strategic | `docs/qa/blueprint-review.md` |
| 3. Blueprint revision (frozen) | strategic | `docs/blueprint.md` |
| 4. Architecture docs | execution | `docs/architecture/*.md` |
| 5. Per-service spec fanout (parallel) | execution | `<svc>/docs/{spark,blueprint,api-contract,data-model,events,build-tasks,acceptance,dependencies}.md` |
| 6. Cross-service consistency | strategic | `docs/qa/cross-service-review.md` |
| 7. Doc registry | deterministic | `docs/doc-registry.md` |
| 8. Verify | deterministic | exit 0 / 1 |

### Model tiers

| Tier | Default | Used for |
|---|---|---|
| `strategic` | `anthropic/claude-opus-4-5` | Blueprint forge, QA review, cross-service review |
| `ensemble` | `[xai/grok-4, deepseek/deepseek-r1]` | Multi-perspective adversarial review |
| `execution` | `deepseek/deepseek-chat` | Per-service docs, architecture docs, research filter |
| `ui` | `anthropic/claude-opus-4-5` | UI/wireframe docs (when enabled) |

Override via `~/.config/architect/config.toml` or env (`ARCHITECT_MODEL_STRATEGIC`, etc.). If a key is missing the router falls back to OpenRouter automatically.

## Per-service docs structure

Every service folder generated by Architect contains exactly:

```
<service>/docs/
  spark.md              # immutable per-service identity
  blueprint.md          # frozen plan slice for this service
  api-contract.md       # endpoints, payloads, errors, auth
  data-model.md         # entities, fields, indexes, ownership
  events.md             # only if blueprint mandates events
  build-tasks.md        # ordered atomic implementation tasks
  acceptance.md         # observable done criteria
  dependencies.md       # in/out runtime + dev deps
```

`architect verify` enforces this whitelist and fails on any extra `.md` files in a service's `docs/` or any `.md` inside `<service>/src/`.

## Brainstorm mode

Pass `--brainstorm` and Architect drives an interactive Q&A in your terminal, following [obra/superpowers'](https://github.com/obra/superpowers) `skills/brainstorming/SKILL.md`. The skill is fetched fresh at runtime (cached for 24h). The session ends with a written, user-approved `spark.md`.

The brainstorm refuses to advance to Phase 1 until the Spark is approved.

## Web research

Pass `--research` and Architect feeds queries through a pluggable search adapter, then runs an 85%-noise filter to keep only implementation-relevant findings. Findings land in `docs/research/findings.md`.

The adapter interface is shaped after [parallel.ai](https://docs.parallel.ai)'s API — Firecrawl is the v1 backend, but a self-hosted Parallel-compatible service is a one-config swap.

## Configuration

```toml
# ~/.config/architect/config.toml

[models]
strategic = "anthropic/claude-opus-4-5"
ensemble  = ["xai/grok-4", "deepseek/deepseek-r1"]
execution = "deepseek/deepseek-chat"
ui        = "anthropic/claude-opus-4-5"
fallback  = "openrouter/auto"

[search]
provider      = "firecrawl"
base_url      = ""           # optional override (e.g. self-hosted Firecrawl)
noise_filter  = 0.85
per_query_cap = 500

[brainstorm]
source    = "github.com/obra/superpowers"
ref       = "main"
cache_ttl = "24h"

[output]
ui_enabled = false
git_init   = true
```

```bash
architect config path           # show file path
architect config show           # dump resolved config
architect config get models.strategic
architect config set search.provider parallel
```

Secrets (API keys) are read from environment **only** — never from the TOML file.

## Verification rules

`architect verify` enforces:

1. Every `.md` under `docs/` matches the root whitelist
2. Every `.md` under `<service>/docs/` matches the per-service whitelist
3. No `.md` under `<service>/src/`
4. `docs/doc-registry.md` exists
