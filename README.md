# Architect

> Idea → frozen Blueprint → strict per-service docs tree.

Architect is a CLI that turns a human-authored product idea into a complete, executable documentation foundation. A high-intelligence model (Claude Opus by default) forges the master Blueprint; a fast-execution tier fans out per-service docs in parallel; a deterministic verifier guarantees no documentation drift.

**Status:** v0.1.0 (pre-release). Core pipeline under active construction.

## Why

Most AI-assisted projects suffer from documentation entropy: scattered READMEs, duplicated explanations, AI-generated essays buried inside source files, scope creep that quietly mutates the original idea. Architect treats the Blueprint as a contract: nothing exists in `docs/` unless the Blueprint mandated it, and every service gets the same eight-document spine.

## Install

```bash
# npm (global)
npm i -g @jamman7/architect

# or bunx (one-shot)
bunx @jamman7/architect new

# or download a single-file binary from
# https://github.com/JAMMAN7-png/Architect/releases
```

## Quickstart

```bash
export ANTHROPIC_API_KEY=…       # or OPENROUTER_API_KEY=… for fallback routing
export FIRECRAWL_API_KEY=…       # only if you pass --research
architect doctor                 # verifies keys, reachability
architect new --brainstorm --out ./my-idea
architect verify --out ./my-idea
```

## Pipeline

| Phase | Model tier | Output |
|---|---|---|
| 0. Spark capture (optional brainstorm) | strategic | `docs/spark.md` |
| 1. Blueprint forge | strategic | `docs/blueprint.draft.md` |
| 2. QA attack review (ensemble) | ensemble + strategic arbiter | `docs/qa/blueprint-review.md` |
| 3. Blueprint revision (frozen) | strategic | `docs/blueprint.md` |
| 4. Service map | strategic | `docs/architecture/service-map.md` |
| 5. Per-service spec fanout (parallel) | execution | `<svc>/docs/{spark,blueprint,api-contract,data-model,events,build-tasks,acceptance,dependencies}.md` |
| 6. Cross-service consistency | strategic | `docs/qa/cross-service-review.md` |
| 7. Doc registry + manifest | deterministic | `docs/doc-registry.md` |
| 8. Verify | deterministic | exit 0/1 |

## Per-service docs structure

Every service folder generated by Architect contains exactly:

```
<service>/docs/
  spark.md              # immutable per-service identity
  blueprint.md          # frozen plan section for this service
  api-contract.md       # endpoints, payloads, errors, auth
  data-model.md         # entities, fields, indexes, ownership
  events.md             # only if blueprint mandates events
  build-tasks.md        # ordered atomic implementation tasks
  acceptance.md         # observable done criteria
  dependencies.md       # in/out runtime + dev deps
```

`architect verify` enforces this whitelist.

## Configuration

Config lives at `~/.config/architect/config.toml` (or `%APPDATA%/architect/config.toml` on Windows). Secrets are read from environment variables only — Architect will refuse to start if any required key is missing and `architect doctor` will tell you exactly what to set.

## Brainstorm mode

Pass `--brainstorm` and Architect runs an interactive Q&A driven by [obra/superpowers'](https://github.com/obra/superpowers) `skills/brainstorming/SKILL.md`, fetched fresh at runtime. The session ends with a written, user-approved `spark.md`.

## Web research

Pass `--research` and Architect feeds queries to a pluggable search adapter. Firecrawl is the v1 backend; the adapter is shaped against the parallel.ai API so swapping in a self-hosted equivalent is one config change.
